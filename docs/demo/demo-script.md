# Demo Script

更新时间：2026-06-09

本文用于演示当前单企业通用后台模板。

## 准备

启动后端和前端：

```bash
docker compose up -d --build
```

基础 seed：

```bash
pnpm run db:seed
```

可选演示数据：

```bash
pnpm run db:seed:demo
```

演示账号密码仅用于本地或测试环境：

```text
manager@example.com  / Demo1234.
operator@example.com / Demo1234.
finance@example.com  / Demo1234.
viewer@example.com   / Demo1234.
```

不要在生产环境执行 demo seed。

## 演示路径

1. 使用管理员账号登录。
2. 打开工作台，说明系统状态、关键资源和常用入口。
3. 打开用户管理，展示用户、角色绑定和状态。
4. 打开角色管理，说明 Role 是岗位 / 职责 / 能力包。
5. 打开权限管理和菜单管理，说明系统内置项由 seed 维护。
6. 打开字典管理和系统参数，说明配置能力。
7. 打开文件中心，展示文件元数据和当前页导出。
8. 打开操作日志、登录日志、系统状态、版本信息、队列状态。
9. 打开消息中心，切换待办 / 通知 / 已处理。
10. 打开审批请求，展示全部相关 / 我的申请 / 待我审批。
11. 新建一个审批请求，审批人为当前管理员。
12. 回到消息中心，确认生成审批待办。
13. 通过审批请求，确认状态变化和通知生成。
14. 点击当前页导出，说明当前导出策略。

## 演示重点

- 当前是单企业后台模板，不是多租户系统。
- Approval Lite 是单步审批，不是流程引擎。
- 当前导出是当前页 CSV，不是全量异步导出。
- S4 知识库和 S6 AI 暂停。
- Department / Position 是未来可选扩展。
