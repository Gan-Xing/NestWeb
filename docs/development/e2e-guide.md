# E2E Guide

更新时间：2026-06-09

Antdpro6 使用 Playwright 做端到端验收。S9 的 E2E 目标是证明当前后台可演示、可交付，而不是覆盖所有边界。

## 环境变量

```bash
E2E_BASE_URL=http://127.0.0.1:8000
E2E_ADMIN_EMAIL=admin@example.com
E2E_ADMIN_PASSWORD=Admin1234.
```

必须使用非生产测试账号。CI 缺少环境变量时可以跳过，但本地完整验收必须提供。

## 当前覆盖

- 登录成功 / 登录失败。
- 刷新保持登录。
- Access token 过期自动 refresh。
- 退出登录。
- 退出后访问受保护页面跳登录。
- Dashboard 和运维页面加载。
- 无权限菜单不可见，直接访问受限页面展示 403。
- 消息中心 tab。
- 通知标记已读 / 全部已读。
- 待办完成。
- 审批请求新建。
- 待我审批 tab。
- 审批通过后状态变化。
- 消息中心和审批请求当前页 CSV 导出。
- 空数据导出提示。

## 新业务页 E2E 建议

1. 管理员可打开页面。
2. 创建一条记录。
3. 搜索能找到该记录。
4. 当前页导出按钮存在。
5. 无权限用户不可见菜单或访问受限。
6. 如果接入消息 / 审批，需要覆盖对应联动。
